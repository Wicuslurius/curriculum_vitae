Publicar/Redireccionar/Obtener

Publicar/Redireccionar/Obtener(PRG) es un patron de diseño de desarrollo web que permite que la pagina que se muestra despues de enviar un formulario se vuelva a cargar, compartir o marcar sin efectos negativos, como enviar el formulario en otro momento. Cuando se envia un formulario web a un servidor a traves de una solictud HTTP POST, los intentos de actualizar la respuesta del servidor pueden hacer que se vuelva a enviar el contenido del POST original, lo que posiblemente provoque resultados no deseados, como una compra web duplicada. Algunos navegadores mitigan eset riesgo advirtiendo al usuario que esta a punto de volver a emitir una solicitud POST. para evitar este problema, muchos desarrolladores web utilizan el patron PRG: en lugar de responder con contenido , el servidor responde a una solicitud POST redirigiendo al cliente a otra ubicacion. la especificacion HTTP 1.1 introdujo el codigo de respuesta HTTP 303 ("Ver otro") para garantizar que , en esta situacion, los navegadores puedan actualizar de forma segura la respuesta del servidor sin que se vuelva a enviar la solicitud POST inicial. El patron PRG no puede abordar todos los escenarios de envio de formularios duplicados. por ejemplo, si un usuario web actualiza antes de que se complete el envio inicial, posiblemente debido a un retraso del servidor , se produce un POST duplicado en ciertos agentes de usuario.
